## Overview

ZeroStarter can be deployed to Vercel for both the frontend (Next.js) and backend (Hono) applications. This guide covers the deployment configuration for both parts of the application.

## Prerequisites

1. A [Vercel](https://vercel.com) account
2. Your repository pushed to GitHub, GitLab, or Bitbucket
3. A PostgreSQL database (e.g., [Neon](https://neon.tech), [Supabase](https://supabase.com), or Railway)

## Frontend Deployment

### Configuration

| Option           | Value                           |
| ---------------- | ------------------------------- |
| Root Directory   | `web/next`                      |
| Framework        | Next.js                         |
| Build Command    | `bun run build` (auto-detected) |
| Output Directory | `.next` (auto-detected)         |

### Environment Variables

Add these environment variables in Vercel project settings:

```
NODE_ENV=production
NEXT_PUBLIC_APP_URL=https://your-frontend-domain.vercel.app
NEXT_PUBLIC_API_URL=https://your-backend-domain.vercel.app

# Optional: Analytics
NEXT_PUBLIC_POSTHOG_KEY=your_posthog_key
NEXT_PUBLIC_POSTHOG_HOST=https://eu.i.posthog.com

# Optional: Feedback
NEXT_PUBLIC_USERJOT_URL=your_userjot_url
```

## Backend Deployment

### Configuration

| Option           | Value           |
| ---------------- | --------------- |
| Root Directory   | `api/hono`      |
| Framework        | Other           |
| Build Command    | `bun run build` |
| Output Directory | `dist`          |

### Environment Variables

Add these environment variables in Vercel project settings:

```
NODE_ENV=production
HONO_APP_URL=https://your-backend-domain.vercel.app
HONO_TRUSTED_ORIGINS=https://your-frontend-domain.vercel.app
BETTER_AUTH_SECRET=your_secret_key
GITHUB_CLIENT_ID=your_github_client_id
GITHUB_CLIENT_SECRET=your_github_client_secret
GOOGLE_CLIENT_ID=your_google_client_id
GOOGLE_CLIENT_SECRET=your_google_client_secret
POSTGRES_URL=your_postgres_connection_string
```

## Deployment Steps

1. **Import Repository**: Go to Vercel dashboard and click "Add New Project"
2. **Select Repository**: Choose your ZeroStarter repository
3. **Configure Project**: Set the root directory to `web/next` for frontend or `api/hono` for backend
4. **Add Environment Variables**: Add all required environment variables
5. **Deploy**: Click "Deploy" and wait for the build to complete

## Custom Domains

After deployment, configure custom domains:

1. Go to your project settings in Vercel
2. Navigate to "Domains"
3. Add your custom domain
4. Update DNS records as instructed
5. Update environment variables with the new domain URLs

## Notes

- Deploy the backend first to get the API URL for the frontend environment variables
- Make sure to update OAuth callback URLs in GitHub/Google with your production domains
- For production, use a managed PostgreSQL service with connection pooling
